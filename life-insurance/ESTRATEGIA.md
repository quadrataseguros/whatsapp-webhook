# Quadrata Seguros — Estratégia de Marketing de Seguro de Vida nas Redes Sociais

## 🎯 PÚBLICO-ALVO DEFINIDO

### Público Primário: O "Provedor de Família"
**Quem é:** Pessoa de 32-50 anos, casada, com 1-3 filhos, que sustenta a casa. Pode ser CLT de cargo médio/alto, profissional liberal ou pequeno empresário.

**Por que ele(a) compra seguro de vida:**
- Tem financiamento imobiliário que a família não pagaria sozinha
- Tem filhos que precisam de escola/faculdade
- Sabe que o INSS não protege o padrão de vida da família
- Já viu ou ouviu falar de alguém que passou por situação difícil

**Onde encontrá-lo:** Instagram (20h-22h), Facebook (grupos de bairro, família), LinkedIn (profissão), WhatsApp (grupos de pais de escola, condomínio)

---

### Público Secundário 1: O Autônomo/MEI
**Quem é:** Profissional liberal (médico, dentista, advogado) ou trabalhador autônomo sem benefícios de emprego. Se parar de trabalhar, para de ganhar.

**Argumento chave:** "Seu salário continua mesmo quando você não pode trabalhar."

---

### Público Secundário 2: O Planejador Patrimonial
**Quem é:** 45-65 anos, tem patrimônio formado, pensa em inventário, herança, ITCMD, sucessão empresarial.

**Argumento chave:** "Seguro de vida é ferramenta de planejamento patrimonial — isento de IR e não entra em inventário."

## 📋 PASSO A PASSO: AS 6 FASES

### FASE 1 — FUNDAÇÃO (Semana 1-2)
**Prioridade máxima. Sem isso, nada funciona.**

- [ ] **1.1** Definir ou criar perfis profissionais nas redes (Instagram @quadrataseguros, LinkedIn, Facebook)
- [ ] **1.2** Bio otimizada: quem somos, o que fazemos, link do WhatsApp
- [ ] **1.3** Foto de capa e identidade visual básica (cores, fontes, logo)
- [ ] **1.4** Definir o número de WhatsApp Business dedicado para leads de seguro de vida
- [ ] **1.5** Configurar respostas automáticas no WhatsApp Business (horário, saudação, ausência)
- [ ] **1.6** Criar o primeiro conteúdo âncora: "O que é seguro de vida e por que você precisa saber" (Reels ou carrossel)
- [ ] **1.7** Configurar o `ANTHROPIC_API_KEY` no arquivo `.env` para usar o gerador

---

### FASE 2 — CONTEÚDO EDUCATIVO (Semana 3-6)
**Objetivo: Ser referência em educação sobre seguro de vida na região/nicho.**

- [ ] **2.1** Publicar 5-6 posts/semana seguindo o calendário de conteúdo
- [ ] **2.2** Focar em quebrar as 5 objeções principais (um post por semana)
- [ ] **2.3** Fazer ao menos 1 Reels educativo por semana (algoritmo prioriza)
- [ ] **2.4** Usar o gerador para criar carrosséis educativos sobre: cobertura, preço, diferenças de seguradoras
- [ ] **2.5** Publicar 3 posts de "autoridade" com dados de Porto Seguro, SulAmérica e Bradesco
- [ ] **2.6** Criar sequência de Stories interativos (enquetes: "Você tem seguro de vida?")

---

### FASE 3 — ENGAJAMENTO E COMUNIDADE (Semana 4-8)
**Objetivo: Criar relacionamento, não só alcance.**

- [ ] **3.1** Responder TODOS os comentários nas primeiras 2 horas (algoritmo recompensa)
- [ ] **3.2** Fazer Stories diários — bastidores, curiosidades, "sabia que..."
- [ ] **3.3** Fazer Lives mensais: "Tira-dúvidas sobre Seguro de Vida" (30 minutos)
- [ ] **3.4** Interagir em grupos de Facebook relevantes (pais, finanças, empreendedores)
- [ ] **3.5** Iniciar parceria com influenciadores locais (conta com 3k-30k seguidores, nicho família/finanças)
- [ ] **3.6** Criar grupo de WhatsApp "Proteção Financeira — Quadrata" para conteúdo exclusivo

---

### FASE 4 — GERAÇÃO DE LEADS (Semana 6-10)
**Objetivo: Converter atenção em leads qualificados.**

- [ ] **4.1** Criar "Isca Digital": PDF gratuito "Guia: Como escolher o seguro de vida certo para sua família"
- [ ] **4.2** Link na bio → Landing page simples com formulário OU direto para WhatsApp
- [ ] **4.3** Stories com "Arraste para cima" (link para WhatsApp) 2x por semana
- [ ] **4.4** Publicar posts de conversão (15% do calendário) com CTA claro
- [ ] **4.5** Criar fluxo de atendimento no WhatsApp: saudação → qualificação → proposta → fechamento
- [ ] **4.6** Configurar a MarIAna (IA do WhatsApp da Quadrata) para qualificar leads de seguro de vida

---

### FASE 5 — TRÁFEGO PAGO (Semana 8-12)
**Só começar quando já tiver consistência orgânica. Tráfego pago amplifica o que já funciona.**

- [ ] **5.1** Impulsionar os 3 posts com melhor engajamento orgânico (R$ 10-20/dia por 7 dias)
- [ ] **5.2** Criar campanha no Meta Ads com objetivo "Mensagens no WhatsApp"
  - Público: 30-55 anos, casados, renda estimada média-alta, interesses: família, finanças, imóvel
  - Excluir: quem já interagiu com a página (já é lead)
- [ ] **5.3** Criar campanha de remarketing para quem visitou o perfil mas não mandou mensagem
- [ ] **5.4** Testar anúncio A/B: abordagem emocional vs educacional (o gerador já faz isso)
- [ ] **5.5** Orçamento inicial sugerido: R$ 600-R$ 1.200/mês em ads

---

### FASE 6 — OTIMIZAÇÃO E ESCALA (Semana 10+)
**Medir, ajustar e escalar o que funciona.**

- [ ] **6.1** Analisar métricas semanalmente: alcance, engajamento, leads, conversão
- [ ] **6.2** Identificar os 3 melhores temas/formatos e criar mais conteúdo deles
- [ ] **6.3** Implementar sistema de CRM para acompanhar leads (pode ser planilha ou Hubspot free)
- [ ] **6.4** Fazer pesquisa com clientes: "Como você nos encontrou?" → ajustar canal
- [ ] **6.5** Expandir para LinkedIn se o público de profissionais liberais/empresários performar
- [ ] **6.6** Criar programa de indicação: cliente que indica ganha desconto ou brinde

## 🚀 COMO USAR O GERADOR DE POSTS

### Via CLI (terminal):
```bash
# Gerar um carrossel educativo para o provedor de família
node life-insurance/post-creator.js --tipo carrossel --persona provedor --pilar educacao

# Gerar roteiro de Reels sobre um tema específico
node life-insurance/post-creator.js --tipo reels --tema "5 mitos sobre seguro de vida"

# Gerar calendário mensal completo
node life-insurance/post-creator.js --calendario

# Gerar variantes A/B de um Reels
node life-insurance/post-creator.js --ab-test --tipo reels --tema "seguro de vida para autônomos"

# Ver todas as opções
node life-insurance/post-creator.js --lista
```

### Via API REST (quando o servidor está rodando):
```bash
# Gerar post
curl -X POST http://localhost:3000/posts/generate \
  -H "Content-Type: application/json" \
  -H "x-secret: SEU_SECRET_AQUI" \
  -d '{"tipo": "carrossel", "persona": "provedor", "pilar": "educacao"}'

# Gerar calendário
curl -X POST http://localhost:3000/posts/calendar \
  -H "Content-Type: application/json" \
  -H "x-secret: SEU_SECRET_AQUI" \
  -d '{"mes": 6, "ano": 2026}'
```
